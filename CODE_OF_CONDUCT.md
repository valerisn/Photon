# Code of Conduct

## Expected Behavior

- Be respectful and direct.
- Keep discussions focused on the project.
- Give useful context when reporting bugs or suggesting changes.
- Assume good intent unless there is a clear reason not to.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Spam or intentionally disruptive comments.
- Publishing private information without permission.
- Posting exploit details for unresolved security issues.

## Enforcement

Project maintainers may remove comments, close issues, reject contributions, or block users who do not follow this code of conduct.

Security concerns should be handled through `SECURITY.md`, not public issue threads.
